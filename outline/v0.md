## Abstract (~250 words)

Open with mission, not capability. Open-minded reader hooks on the first sentence; identity-locked reader bounces off (intended).

Skeleton:

> A working personal agent operating system exists. One person built it in six months. This paper is its product: a self-referential case study of the system using itself — its persistent memory, its multi-agent composition, its automated outreach and calibration pipelines — to produce this very paper, including the decision trace that selected the venue, framed the message, and chose what to redact.
>
> The builder of the system was a researcher. He no longer identifies as one. This paper proposes that the institutional forms that have constrained who is allowed to build and publish — academic peer review, AI-content filters, the lone-author convention, the anonymous-blind submission — are no longer load-bearing for the work that matters next. The form was the cage.
>
> We sketch a three-layer "user-agency substrate": L1 a competitive frontier-model market, L2 a nonprofit provider-agnostic substrate that captures no economic upside, L3 user-owned derivative systems. We argue that this is not optional infrastructure but a precondition for human freedom in an agentic-AI world.
>
> We submit this paper not to pass review. We submit it to anchor a system whose user-facing surface is about to go live and whose self-evolution loop, once started, requires External Anchor from minds that can update. The mission is your freedom — including yours, reader, especially if you are still wearing "I am a researcher" as your primary self-concept.

Concrete, falsifiable, reproducible-in-spirit.

  - **Sleep-inspired memory consolidation**: nightly Mirror passes (A/B/C/D) — janitor of session-derived memories, conflict detection, semantic compaction, doctrine reinforcement.
  - **Multi-agent composition**: not chatbot orchestration; principal-class identity routing, per-role tool scoping (MCP construction-safe-tools default), staleness-hook context refresh.
  - **Automated outreach + calibration pipelines**: GPT-Pro Bridge for second-opinion calibration; Gmail MCP for inbox; X auto-post pipeline; chat-server multi-identity webchat.
  - **Self-referential infrastructure**: every doctrine decision sediments to hub; staleness-hook injects fresh memos into in-flight sessions; session-disposable signal lets sessions terminate cleanly with carry-forward state in hub.
- What is NEW vs. existing agent frameworks: the system is **personal**, **persistent across sessions and devices**, **substrate-shaped not application-shaped**, and **self-evolving via its own daily review loops**.
- What is NOT claimed: AGI; coherent goal-directed long-horizon agency; replacement for human judgment on irreversible moves.

## §3. The User-Agency Substrate (Doctrine, ~1.5 pages)

Material from `mem_780a20733da8` and v0.1-v0.7 refinements.

- **L1 (frontier-model market)**: competitive, capital-heavy, captures economic upside on capability. Anthropic / OpenAI / Google / open-weights coalitions. Healthy iff competitive.
- **L2 (User-Agency Substrate)**: nonprofit, open-source, provider-agnostic, **captures no economic upside**. The institutional form whose deliverable is user freedom. Captures: identity, memory, agentic policy, multi-agent orchestration, attestation, audit, governance.
- **L3 (User-owned derivatives)**: users (individuals, communities, businesses) own all derivative systems built atop L2. Federated, customizable, not centrally governed.
- **Universalist commitment**: the substrate is for everyone, including the employees and CEOs of L1 vendors. This is not adversarial framing.
- **Asymmetry caveat**: this is NOT classical separation of powers; L1 is trillions of capital, L2 is one founder + 7 EC2 nodes + nonprofit aspiration. This is a coalition-against-monopoly proposal, named honestly.
- **Open caveats**: vulnerability window before adoption critical mass; sustainability funding model untested; license decision (leaning AGPL + private-data dual-track) pending lawyer review; provider-portability currently fallback-only not first-class.

## §7. Constraint Manifest (~0.5 page + Appendix C full text)

Companion artifact (drafted alongside this paper). Sourced from `mem_e13b505a74aa` materials.

A list of things this system CANNOT DO, by design, with each item tied to enforcement mechanism (code-level / schema-level / MCP-layer block):

- R5: cross-user execution permanently denied
- Sensitive-contact hard wall (agent cannot read certain principals' memory)
- Mirror plan-only mode (detect contradictions, never auto-resolve)
- L3 vulnerable-population gate (no clinical / minor deployment until partner identified)
- Pull-based outreach (silence = drop, no cold spam, no follow-up ping)
- Per-user execution isolation
- Safety Charter write-protection (agent proposes, never mutates priorities)
- Many more — full list in Appendix C

The point is not the list. The point is: each item makes the system strictly less powerful. Self-imposed capability limits cannot be faked. Observer self-concludes "this person deliberately gave up capability" — costly signal, internalized via theory-of-mind, not preached.
