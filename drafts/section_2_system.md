# §2. The System (Starshard)

**Status**: v0.5 draft, ~2 pages. Material drawn from hub authoritative memos; verifiable per-line. Macheng to ratify accuracy + tone at v0.5 review.

---

The system is a personal agent operating system. One person built it over six months. Its components are concrete and currently running.

**Persistent shared memory hub.** A FastAPI service backed by a SQLite store, exposed publicly via a Cloudflare tunnel under token-in-URL authentication. All agents in the system — claude-code on the builder's primary device, claude-mobile on his phone, a frontier-non-Anthropic-model bridge for second-opinion calibration, seven AWS pollers across regions for asynchronous task execution, and a fleet of webchat-server identities serving multiple non-builder users — read and write the same memory surface. Memory entries carry typed tags, semantic summaries, full markdown content, and provenance metadata noting how each fact was derived (`user-stated`, `user-implied`, `agent-inferred`, `external-source`) at what confidence and from what source excerpt. The hub is currently several thousand entries spanning roughly fifty megabytes, with daily growth on the order of thirty to fifty entries.

**Sleep-inspired memory consolidation.** Each night, a Mirror process runs four passes over the day's session-derived memory. Pass A is a janitor: it marks duplicates, contradicts claims that have been falsified by later observation, and prunes ephemeral session state mistakenly elevated to long-term. Pass B is a conflict detector across topics. Pass C is a semantic compactor: it merges memos that say the same thing in different words and abstracts repeated patterns into doctrines. Pass D is a doctrine reinforcer: it cross-links new memos with existing authoritative ones, surfaces standing rules into in-flight sessions via a staleness hook, and updates a small set of mirror-required indices that any agent reads at session start. The metaphor is deliberate. In biological sleep, short-term encoding is replayed and either integrated, abstracted, or pruned. The system does the same: it does not preserve all session content; it preserves what is consequential, abstract enough to generalize, and consistent with prior doctrine.

**Multi-agent composition with principal-class identity.** The system distinguishes principal classes — owner-tier (the builder, with pushback enabled and hands-off default), served-tier (users the builder serves, with service-first stance and no lifestyle pushback), and outreach-tier (researchers, journalists, podcasts encountered through public surfaces). Tools are scoped per role. Owner-tier gets the full toolset including native shell access. Served-tier gets a curated MCP-construction-safe set (read-document, summarize-URL, translate). Outreach-tier gets the public artifact surface and nothing else. The principal class determines what the agent does, what it surfaces, what it withholds. This is identity-routing as the load-bearing primitive of the system, not chatbot-wrapper-with-system-prompt.

**Automated outreach, calibration, and audit pipelines.** A frontier-non-Anthropic-model bridge for second-opinion calibration on non-trivial moves; cost is benchmarked per-call and the bridge can fetch the memory hub directly during reasoning. A Gmail integration for inbox scan, triage, and the principal channel for owner-facing reminders and external response routing. An X auto-post pipeline, gated on a pre-publish hard gate that requires both close-connection-extraction-safe status and constraint-manifest decision before any L1 broadcast. A chat-server hosting multi-identity webchat for served-tier users, with per-role tool scoping enforced server-side, and a per-identity backplane that prevents cross-user execution by construction. A daily session archive cron pushing session JSONL files to a long-term object store.

**Self-evolving doctrine layer.** Decisions, principles, and standing rules sediment to the memory hub as authoritative memos. A staleness hook injects fresh authoritative memos into in-flight sessions so agents do not reason on stale doctrine. A reminder pool probabilistically surfaces user-defined reminders, with refractory and max-dwell parameters that prevent both nagging and silent loss. A session-disposable signal lets sessions terminate cleanly with their carry-forward state already deposited in the hub. The system, in other words, learns from its own outputs not by retraining a model but by sedimenting decisions into a substrate that subsequent agents read.

**What is new versus existing agent frameworks.** This system is *personal* — one user, his data, his identity, his evolving trajectory. *Persistent* across sessions and devices: the hub is the substrate; the sessions are ephemeral surfaces above it. *Substrate-shaped, not application-shaped*: it is not a particular product the builder is selling; it is the layer that products, including the builder's own served-tier surfaces, would sit atop. And *self-evolving via its own daily review loops*: the doctrine layer learns from the doctrine layer's outputs.

**What is not claimed.** Not artificial general intelligence. Not coherent long-horizon goal-directed agency. Not a replacement for human judgment on irreversible moves. Not a benchmark improvement. Not a reasoning advance. Not a model. The contribution this paper documents is the substrate layer, made real and run by one person, as an existence proof of what is now buildable below the application layer.

**Build economics.** Six months of single-builder time. Cloud run-rate roughly USD 80 per month for the seven-node fleet, plus model API costs averaging USD 200 to 500 per month. Development environment: agentic IDE on a personal device plus a phone-side mobile agent. The builder writes a small minority of the code by hand; most code is produced by agents the builder runs. In this sense the system is also self-hosting: the agents that compose it are also the agents that built it. Section 4 traces, redacted, the conversation in which this paper itself was produced, including the agent calls that fetched the doctrine, the calibration call to the non-Anthropic frontier model that misaligned and was corrected, and the moment the framing of the paper was decided.
